git入门

2018-07-31 / 3 min read

起步

首先到官网下载一个git，我这里以win10为例。
https://git-scm.com/downloads
下载好了直接下一步就行了，安装完了鼠标右键菜单会出现以下两个选项：

配置

安装完git后，直接在桌面右键选择 Git Bash Here，进入git终端，进行配置。

  1. 添加用户名和用户邮箱
git config --global user.name "your GitHub username"
git config --global user.email "your GitHub Email"

ps：输入这两个命令时，是没有提示的。

  1. 查看用户是否添加成功
git config -l
  1. ssh配置
  • 添加GitHub注册邮箱到ssh,连续三次回车就行了，成功会出现一个很多泡泡的信息
ssh-keygen -t rsa -C "youremail@domain.com"
  • 输入 eval "$(ssh-agent -s)", 添加秘钥到 ssh-agent
  • 再次输入 ssh-add ~/.ssh/id_rsa, 添加生成的SSH Key到ssh-agent
  • 用vim打开id_rsa.pub文件，将里面的内容全部复制
  • vim全部复制命令： 按 esc键，然后输入ggyG
  • vim退出文档编辑： 按esc键，然后输入:q

如果觉得 vim 不好操作,可以直接到 C盘找到文件

一般都是以上的路径

  1. GitHub ssh配置
  • 配置ssh的基本流程可以按着下面的步骤进行
  • 完成以上步骤就算是基本配置完了，可以愉快的进行代码提交了。

常用命令

这里以一个简单的小demo为例来进行介绍git中常用的命令。

  1. 新建一个文件夹，就是你的项目
  2. 在里面新建一个名为 README/md格式的文件，在里面输入一句简单的话来测试一下。
  3. 鼠标右键菜单Git Bash Here输入一下代码创建本地仓库
git init

创建成功后会多出一个master本地分支来

  1. 首先查看一下本地文件的状态，如果未提交的话文件会显示红色。
git status
  1. 添加要提交的文件，输入一下代码
git add READM.md   // 添加单个文件，直接添加文件名就行
git add .    // 这个命令简单点来说就是直接添加全部，一般都用这个
  1. 提交刚刚添加的文件
git commit -m "first commit"
  1. 提交完之后再来查看一下状态
  1. 然后就到GitHub上创建一个新的仓库，步骤如下
  1. 添加远程仓库地址
git remote add origin git@github.com:qq1332783374/git_demo.git
  1. 然后就是提交到GitHub上面去了
git push -u origin master    // 初次提交要加上 -u，下一次提交就不用了
  1. 提交成功后就可以在 GitHub上面查看，刚刚所提交的内容。记得刷新一下页面。
  1. 以上就差不多是一个git的基本操作流程了，用户只需配置一次就行了，配置完跟着命令敲应该是没问题的。

如有错误的地方，请大家指出。谢谢！欢迎大家多多交流和学习！
