02-React笔记 React中的虚拟DOM

· 2019-06-16 ·

1. React 虚拟DOM 原理

1.1 真实DOM版本

  1. state 数据
  2. JSX 模板
  3. 数据 + 模板 结合，生成真实的DOM，显示到页面
  4. state 发生改变
  5. 数据 + 模板 结合，生成真实的DOM，替换原始DOM再重新显示到页面

以上是React 虚拟DOM实现的一个最底层的原理，同时这样也存在着大量的缺陷，就是一个性能消耗的问题，每次都要重新生成和替换。例如说头部和底部的公共组件不怎么变的，就显得没那么必要。

1.2 真实DOM对比版本

  1. state 数据
  2. JSX 数据
  3. 数据 + 模板 结合，生成真实DOM，显示到页面
  4. state 发生改变
  5. 数据 + 模板结合，生成真实DOM，不直接替换原始DOM
  6. 通过 DocumentFragment和原始DOM做对比，找差异
  7. 找出发生改变的DOM节点，然后操作相应的DOM节点

以上这个版本的确是提升了不少的性能，但同时也消耗了一些性能。在对比差异的时候，也消耗了一部分的性能。这个的版本性能提升并不明显，所以得继续升级。

1.3 虚拟DOM节点版本

先区分一下真实DOM和虚拟DOM的区别

真实DOM: 是由JavaScript去调用 WebApi ，由于不是JavaScript本身的API这一过程会存在大量的性能消耗。

<div id="app">
	<span>Hello React</span>
</div>

虚拟DOM: 其实就是一个JavaScript对象，一个虚拟DOM就相当于是创建一个JavaScript的对象，不存在很多的性能消耗问题

['div', {id: 'app'}, ['span', {}, 'Hello React']]

虚拟DOM的版本可以这样子来实现:

  1. state 数据
  2. JSX 模板
  3. 数据 + 模板 生成虚拟DOM，用来描述真的DOM
  4. 用虚拟DOM的结构生成真实的DOM，来显示
  5. state 发生改变
  6. 数据 + 模板 生成新的虚拟DOM
  7. 比较原始的虚拟DOM和新的虚拟DOM，找出区别
  8. 操作DOM，改变DOM内容

过程： JSX -> createElement -> 虚拟DOM JS 对象 -> 真的DOM

这一个版本的实现是进行js原生的对象对比，不用直接调用webApi来进行操作。相对来说性能提升了好几倍，甚至不止。

以上就是一个虚拟DOM的原理，本质上就是一个js对象。不怎么消耗性能
